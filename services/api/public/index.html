01

OS-level protection

Cloak uses setContentProtection — documented OS-level capture protection on macOS and Windows. Not a filter. Not a trick.

02

Works before pixels render

The exclusion happens at the compositor layer before any recorder sees a frame. OBS, Zoom, Teams, Meet, and Loom all receive a feed with the overlay simply not there.

03

You still see it normally

Your physical display renders the overlay at full opacity. Only capture clients are excluded. You read naturally — the recording shows nothing unusual.

macOS install command

Paste this in Terminal. It downloads the DMG, installs Cloak into Applications, clears the quarantine flag, and opens the app.

set -e
DMG="$HOME/Downloads/Cloak-1.0.1-arm64.dmg"
MOUNT_DIR="$(mktemp -d)"
curl -L "https://cloak-flax.vercel.app/v1/download/darwin" -o "$DMG"
hdiutil attach "$DMG" -mountpoint "$MOUNT_DIR" -nobrowse -quiet
rm -rf /Applications/Cloak.app
cp -R "$MOUNT_DIR/Cloak.app" /Applications/
hdiutil detach "$MOUNT_DIR" -quiet
rmdir "$MOUNT_DIR"
xattr -cr /Applications/Cloak.app
open /Applications/Cloak.app

⚠️ Install notes

macOS: The command above already runs xattr -cr /Applications/Cloak.app after installing the app.

Windows: The current Windows build is unsigned, so SmartScreen may warn before launch. Use it for testing only until signed release builds are available.

Run xattr -cr /Applications/Cloak.app in Terminal, then open Cloak normally. This clears the quarantine flag macOS sets on apps downloaded outside the App Store. Cloak isn't signed with an Apple Developer certificate yet — the warning is a Gatekeeper check, not a real problem with the app.

Cloak uses setContentProtection, a documented Electron API backed by macOS and Windows capture-protection APIs. It works with most recorders that use the system capture pipeline — OBS, Zoom, Teams, Meet, Loom, QuickTime. Apps using newer ScreenCaptureKit on macOS 14+ may be able to see it. Always test with your exact setup before relying on it for critical recordings.
